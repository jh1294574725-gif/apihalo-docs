# 控制台（入门）

更新时间：2026-04-15

首次接入建议按下面顺序操作。

## 第一步：登录控制台

访问：

`https://apihalo.com/`

登录后进入控制台。

## 第二步：创建 API Key

在控制台创建一个新的 `API Key`，创建完成后立即复制保存。

建议：

- 按业务、环境或项目拆分不同 Key
- 测试环境和生产环境不要共用同一个 Key
- 如果怀疑泄露，请立即删除旧 Key 并重建

## 第三步：确认 Base URL

默认 OpenAI 兼容地址：

```text
https://apihalo.com/v1
```

如果你的 SDK 或工具会自动补 `/v1`，请改填：

```text
https://apihalo.com
```

## 第四步：先获取模型列表

先看当前账号真实可用模型：

```bash
curl https://apihalo.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

重点：

- 后续请求里的 `model` 必须从这里面选
- 不要直接照搬别的平台模型名

## 第五步：做一次最小可用验证

```bash
curl https://apihalo.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_MODEL_ID",
    "messages": [
      {"role": "user", "content": "你好"}
    ]
  }'
```

## 第六步：再按需开启专项能力

如果你还需要以下能力，再继续看对应栏目页：

- 向量检索
- 图像生成
- 音频识别与合成
- 视频生成
- Midjourney
- Suno
- 搜索阅读
- OCR 文档解析

## 常见误区

- 先写业务逻辑，最后才看 `/v1/models`
- 把上游平台的文档地址直接发给客户
- 默认认为所有文本模型都支持 `tools`、结构化输出、联网搜索
