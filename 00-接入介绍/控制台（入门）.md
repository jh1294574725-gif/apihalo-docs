# 控制台（入门）

更新时间：2026-04-15

首次接入建议按下面顺序操作。本页的目标是帮助调用方在 5 分钟内完成一次最小可用验证。

## 一、进入控制台

访问：

`https://apihalo.com/`

登录后进入控制台。

## 二、创建 API Key

在控制台创建一个新的 `API Key`，创建完成后立即复制保存。

建议做法：

- 按业务、环境或项目拆分不同 Key
- 测试环境与生产环境不要共用同一个 Key
- 如果怀疑泄露，请立即删除旧 Key 并重建

## 三、确认 Base URL

默认 OpenAI 兼容地址：

```text
https://apihalo.com/v1
```

如果 SDK、插件或客户端会自动补 `/v1`，则填写：

```text
https://apihalo.com
```

## 四、先获取模型列表

先查看当前 Key 下真实可用模型：

```bash
curl https://apihalo.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

这一步的作用：

- 确认当前 Key 是否可用
- 确认当前模型是否已经开放
- 后续请求里的 `model` 必须从这里选择

## 五、完成一次最小可用验证

推荐先用标准聊天接口验证：

```bash
curl https://apihalo.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_MODEL_ID",
    "messages": [
      {"role": "user", "content": "你好"}
    ]
  }'
```

如果可以正常返回内容，说明以下 4 项都已经打通：

- Key 可用
- Base URL 正确
- 模型可见
- OpenAI 兼容主线已通

## 六、再按需开启专项能力

如果还需要以下能力，再继续看对应栏目页：

- 向量检索
- 图像生成
- 音频识别与合成
- 视频生成
- Midjourney
- Suno
- 搜索阅读
- OCR 文档解析
- 开发工具接入

## 七、新手最容易踩的误区

- 先写业务代码，最后才检查 `/v1/models`
- 直接照搬其他平台的模型名或 Base URL
- 默认认为所有文本模型都支持联网搜索、结构化输出、工具调用
