# 平台简介

更新时间：2026-04-15

ApiHalo 是面向开发者的统一 AI API 接入平台，站点地址为：

`https://apihalo.com/`

对于调用方来说，业务真正接入的是 ApiHalo 对外公开的统一接口。

## 一、ApiHalo 是什么

ApiHalo 的作用是把不同模型能力统一为稳定的对外接入方式。

对接时，调用方只需要关注：

- ApiHalo 的 `API Key`
- ApiHalo 的 `Base URL`
- ApiHalo 当前实际开放的模型与能力

这意味着：

- 不需要直接填写其他平台的原始地址
- 不需要直接依赖其他平台的原始鉴权方式
- 模型与能力判断都以 ApiHalo 当前返回结果为准

## 二、接入时只认 3 个核心参数

| 参数 | 说明 | 获取方式 |
| --- | --- | --- |
| `API Key` | 用于接口鉴权 | 在控制台创建 |
| `Base URL` | 用于确定请求入口 | 默认 `https://apihalo.com/v1` |
| `model` | 用于指定调用模型 | 先调用 `/v1/models` 获取 |

## 三、默认公开接入原则

为了保证兼容性稳定、迁移成本可控、后续更换上游时客户端无需重写逻辑，ApiHalo 对外默认统一采用 `OpenAI Compatible` 接口口径。

对于大多数客户，最推荐的公开接入方式只有两步：

1. `GET https://apihalo.com/v1/models`
2. `POST https://apihalo.com/v1/chat/completions`

如果是开发工具、自定义 Provider 或 Codex 一类场景，再额外验证：

3. `POST https://apihalo.com/v1/responses`

## 四、适用范围

本套文档适用于以下接入方式：

- 自研服务端或中台系统接入
- OpenAI SDK 接入
- 支持 `OpenAI` 或 `OpenAI Compatible` 的第三方工具
- 文本、代码、多模态、向量、图像、音频、视频等能力接入

## 五、接入时最容易踩错的 3 件事

1. 没有先看 `/v1/models`，直接把模型名写死
2. 把其他平台的模型总数，误认为当前 Key 真实可用模型总数
3. 默认认为所有模型都支持 `tools`、结构化输出、联网搜索、文件理解、视频生成等全部高级能力
