# ApiHalo 接入文档

ApiHalo 提供统一的 AI API 接入能力。对于调用方来说，业务真正对接的是 ApiHalo，而不是其他平台的原始地址。

## 二、接入只要记住 3 条规则

| 规则 | 说明 |
| --- | --- |
| 可用模型以 `/v1/models` 为准 | 不同 Key、分组、账号、能力范围下，模型可见性可能不同 |
| 始终使用 ApiHalo 的参数 | 接入时使用 ApiHalo 的 `API Key`、`Base URL`，不要直接套用其他平台的接入地址 |
| 默认优先走 OpenAI Compatible | 绝大多数 SDK、客户端、插件、第三方工具都优先按 OpenAI 兼容方式接入 |

## 三、默认推荐接法

对于大多数客户，最稳的公开接入流程如下：

1. `GET https://apihalo.com/v1/models`
2. 从返回结果中选择当前 Key 可用模型
3. `POST https://apihalo.com/v1/chat/completions`

如果是开发工具或自定义 Provider 场景，再继续验证：

4. `POST https://apihalo.com/v1/responses`
