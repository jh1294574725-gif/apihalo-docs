# 常见问题

更新时间：2026-04-15

本页汇总首次接入中最常见的问题。建议和 [Models（列出模型）](03-进阶与系统接口/Models（列出模型）.md)、[支持矩阵](03-进阶与系统接口/支持矩阵.md) 一起阅读。

## 一、模型相关问题

### 1. 为什么不能直接照搬其他平台的模型列表

因为调用方实际接入的是 ApiHalo，而不是其他平台的原始接口。

请始终以 `GET https://apihalo.com/v1/models` 的返回结果为准。

### 2. 为什么文档一直强调先调 `/v1/models`

因为不同账号、分组、套餐、能力开通范围、Key 可见范围都可能不同。

同一个模型在一个 Key 下可见，不代表在另一个 Key 下也可见。

### 3. 后续模型有变化时要改代码吗

通常不用重写整套逻辑。

前提是业务继续使用：

- ApiHalo 的 `API Key`
- ApiHalo 的 `Base URL`
- ApiHalo 返回的真实模型 ID

## 二、接口相关问题

### 4. 最稳的接入方式是什么

优先使用：

- `GET /v1/models`
- `POST /v1/chat/completions`

大多数文本、代码、多模态理解场景都应先从这两步开始。

### 5. 文件分析是不是一定要用 `/v1/files`

不是。

公开接入时优先使用 `POST /v1/chat/completions` 的多模态输入，把文件内容作为 `file` 类型传入。

### 6. 图片、音频、视频是不是所有模型都支持

不是。

这类能力都属于按模型开放或按能力开放，请先查看：

- `GET /v1/models`
- [支持矩阵](03-进阶与系统接口/支持矩阵.md)

## 三、配置相关问题

### 7. Base URL 到底填 `https://apihalo.com` 还是 `https://apihalo.com/v1`

默认填写：

```text
https://apihalo.com/v1
```

如果 SDK、插件或客户端会自动补 `/v1`，则填写：

```text
https://apihalo.com
```

### 8. 为什么返回 401

常见原因：

- `API Key` 写错
- `Bearer` 头丢失
- Key 已删除、已禁用或已过期
- 使用了其他平台的 Key

### 9. 为什么返回 404 或 `model_not_found`

常见原因：

- 模型 ID 拼错
- 模型不在当前 Key 的可见范围内
- 直接使用了其他平台文档里的模型名

### 10. 为什么返回 429

通常表示限流、速率限制或短时间请求过多。

建议：

- 降低并发
- 增加指数退避重试
- 检查是否持续发送无效请求

## 四、任务与异步问题

### 11. 视频生成为什么不是立刻返回结果

视频生成通常是异步任务。

标准流程是：

1. `POST /v1/videos` 或专项视频接口
2. 记录返回的 `task_id`
3. `GET /v1/videos/{task_id}` 查询状态
4. 成功后再获取内容地址
