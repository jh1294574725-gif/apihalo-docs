# 其他工具

更新时间：2026-04-15

如果某个工具没有单独列在本目录，但它支持 `OpenAI`、`OpenAI Compatible`、`Custom OpenAI` 或 `OpenAI-style`，通常都可以按同一套方式接入。

## 先判断工具属于哪一类

### 第一类：标准 OpenAI Compatible 工具

这类工具通常直接调用：

- `POST /v1/chat/completions`

常见做法：

- 选择 `OpenAI`
- 选择 `OpenAI Compatible`
- 选择 `Custom OpenAI`

### 第二类：要求 `Responses API` 的工具

这类工具会直接调用：

- `POST /v1/responses`

例如需要自定义 Provider 的部分代理式开发工具。

这类场景建议先对目标模型做一次 `/v1/responses` 测试，再写入工具配置。

## 通用填写模板

- Provider：`OpenAI` / `OpenAI Compatible`
- Base URL：`https://apihalo.com/v1`
- API Key：ApiHalo API Key
- Model：从 `/v1/models` 返回结果中选择

## 通用排查顺序

1. 先确认工具是否支持自定义 Base URL
2. 再确认是否自动拼接 `/v1`
3. 再确认模型名是否来自 `/v1/models`
4. 如果工具要求 `Responses API`，先验证 `/v1/responses`
5. 最后再排查是否是模型能力未开通

## 最稳的建议

如果某个工具接入失败，优先先用 `curl` 或 OpenAI SDK 验证连通性。

对于标准 OpenAI Compatible 工具，优先验证 `POST /v1/chat/completions`。

对于要求 `Responses API` 的工具，优先验证 `POST /v1/responses`。
