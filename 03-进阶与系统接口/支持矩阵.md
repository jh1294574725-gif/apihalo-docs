# 支持矩阵

更新时间：2026-04-15

## 使用说明

这份矩阵用于帮助你判断：

- 应该优先调用哪个接口
- 某类能力是默认支持还是按模型开放
- 是否需要先做能力验证

## 接口支持矩阵

| 场景 | 接口 | 可用范围 | 说明 |
| --- | --- | --- | --- |
| 获取可用模型 | `GET /v1/models` | 默认支持 | 接入第一步，所有模型选择都以此结果为准 |
| 标准文本对话 | `POST /v1/chat/completions` | 默认支持 | 绝大多数聊天与生成场景优先使用 |
| 流式输出 | `POST /v1/chat/completions` + `stream=true` | 默认支持 | 适合实时输出 |
| 标准补全 | `POST /v1/completions` | 兼容支持 | 主要用于兼容旧客户端 |
| 函数调用 | `POST /v1/chat/completions` | 按模型开放 | 不要默认所有文本模型都支持 |
| 结构化输出 | `POST /v1/chat/completions` | 按模型开放 | JSON 输出能力需要单独验证 |
| 图片 / PDF / 文件理解 | `POST /v1/chat/completions` | 按模型开放 | 公开接入优先按多模态输入处理 |
| 音频理解 | `POST /v1/chat/completions` | 按模型开放 | 适合语音问答与音频理解 |
| 视频理解 | `POST /v1/chat/completions` | 按模型开放 | 适合视频理解场景 |
| Responses API | `POST /v1/responses` | 部分模型可用 | 客户端明确要求时再启用 |
| 向量嵌入 | `POST /v1/embeddings` | 按模型开放 | 适用于检索和知识库 |
| 图像生成 | `POST /v1/images/generations` | 按模型开放 | 仅对已开放图像模型可用 |
| 音频转写 | `POST /v1/audio/transcriptions` | 按模型开放 | 仅对已开放音频模型可用 |
| 音频翻译 | `POST /v1/audio/translations` | 按模型开放 | 仅对已开放音频模型可用 |
| 语音合成 | `POST /v1/audio/speech` | 按模型开放 | 仅对已开放音频模型可用 |
| 视频生成任务创建 | `POST /v1/videos` | 按模型开放 | 标准 OpenAI 兼容视频入口 |
| 视频任务查询 | `GET /v1/videos/{task_id}` | 按模型开放 | 用于查询状态 |
| 视频内容获取 | `GET /v1/videos/{task_id}/content` | 按模型开放 | 用于获取最终视频 |
| 重排序 | `POST /v1/rerank` | 按模型开放 | 适用于搜索和召回排序 |
| 文件管理接口 | `/v1/files` | 当前不作为公开接入前提 | 不建议客户依赖 |

## 客户侧推荐接入顺序

1. 先接 `GET /v1/models`
2. 再接 `POST /v1/chat/completions`
3. 如需流式输出，再加 `stream=true`
4. 如需文件、PDF、图片、音频、视频理解，优先继续走 `chat/completions`
5. 如需专项能力，再逐项接 `embeddings`、`images`、`audio`、`videos`、`rerank`

## 结论

默认情况下，先把 OpenAI 兼容主线接好，再按能力逐项增加，是最稳的方案。
