# CODE&错误码

更新时间：2026-04-15

当请求失败时，优先查看 HTTP 状态码和返回体里的错误结构。本页用于帮助调用方快速定位常见报错原因。

## 一、通用错误结构

OpenAI 兼容接口通常返回：

```json
{
  "error": {
    "message": "错误信息",
    "type": "new_api_error",
    "code": "具体错误码"
  }
}
```

## 二、常见 HTTP 状态

| HTTP 状态 | 常见含义 |
| --- | --- |
| `400` | 请求参数错误、字段缺失、请求体格式不正确 |
| `401` | Key 错误、Key 无效、未携带鉴权头 |
| `403` | 没有权限访问该能力，或当前 Key / 账号受限 |
| `404` | 路径写错、Base URL 错误、把具体接口路径误填成 Base URL |
| `429` | 请求频率过高、触发速率限制 |
| `500 / 502 / 503` | 临时错误、临时不可用、服务处理异常 |

## 三、常见错误码

| 错误码 | 常见含义 |
| --- | --- |
| `model_not_found` | 模型不存在、模型未开放、模型名错误 |
| `access_denied` | 没有权限访问该模型或该能力 |
| `bad_request_body` | 请求体结构错误或字段不符合要求 |
| `insufficient_user_quota` | 额度不足 |
| `read_request_body_failed` | 请求体读取失败 |
| `rate_limit_check_failed` | 触发频率限制或限流检查失败 |

## 四、最常见的排查顺序

1. 先检查 `API Key`
2. 再检查 Base URL
3. 再检查模型名是否来自 `/v1/models`
4. 再检查请求体字段是否符合当前接口
5. 最后再看是否触发了能力限制、额度不足或限流

## 五、特别提醒

如果看到 `model_not_found`，不要第一时间怀疑平台故障，先确认：

- 模型是否来自 ApiHalo 的 `/v1/models`
- 当前 Key 是否确实有权限访问该模型
- 是否误用了其他平台文档中的模型名
